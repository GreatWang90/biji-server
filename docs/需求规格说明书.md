# 比鸡 - 多人在线对战卡牌游戏 需求规格说明书

**文档版本**: v1.0
**编写日期**: 2026-03-06
**项目状态**: 已完成（可持续迭代）

---

## 1. 项目概述

### 1.1 项目背景

"比鸡"是一款源自中国民间的扑克牌游戏，玩家将9张手牌分为头墩、中墩、尾墩三组（各3张），与其他玩家逐墩比较大小，以赢取金币。本项目将传统线下玩法搬至线上，支持多人局域网/互联网实时对战。

### 1.2 项目目标

- 实现完整的比鸡游戏玩法，支持2-6人在线对战
- 提供流畅的 Web 端游戏体验，无需安装客户端
- 支持局域网和互联网部署
- 服务端发牌，防止作弊

### 1.3 目标用户

- 休闲娱乐玩家
- 局域网内多人聚会场景
- 对传统牌类游戏感兴趣的用户

## 2. 功能需求

### 2.1 用户系统

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 用户注册 | 用户名(>=2字符) + 密码(>=3字符)，自动分配随机头像 | P0 |
| 用户登录 | 用户名 + 密码验证 | P0 |
| 初始金币 | 新用户注册获得5000金币 | P0 |
| 用户头像 | 6个预设SVG卡通头像，注册时随机分配 | P1 |

### 2.2 房间系统

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 创建房间 | 设置人数(2-6人)、底分(1-100)、摆牌时间(30-120秒)、房间密码 | P0 |
| 加入房间 | 输入房间号 + 密码加入 | P0 |
| 离开房间 | 任意时刻可退出房间 | P0 |
| 准备/取消准备 | 玩家进入房间后切换准备状态 | P0 |
| 自动开始 | 全部玩家准备后1.5秒自动开始游戏 | P0 |
| 房主转移 | 房主离开时自动转移给下一位玩家 | P1 |
| 空房清理 | 所有玩家离开后自动删除房间 | P1 |

### 2.3 游戏核心玩法

#### 2.3.1 发牌

- 服务端洗牌发牌（防作弊）
- 每人发9张牌
- 2-5人使用52张牌，6人使用54张牌（含大小王）
- 发牌后手牌按点数降序排列

#### 2.3.2 摆牌阶段

- 玩家将9张手牌分为三墩，每墩3张：
  - **头墩**: 第一组3张
  - **中墩**: 第二组3张
  - **尾墩**: 第三组3张
- 限时摆牌（房间设置的倒计时）
- 超时未摆牌自动按手牌顺序摆放
- 支持手动拖拽/点击放置
- **AI智能摆牌**: 一键自动优化摆牌（穷举所有组合找最优解）
- 确认摆牌后展示只读结果，不可修改

#### 2.3.3 牌型等级（从高到低）

| 牌型 | 描述 | 示例 |
|------|------|------|
| 豹子(三条) | 三张相同点数 | AAA |
| 同花顺 | 三张同花色且连续 | 黑桃JQK |
| 同花 | 三张同花色 | 红心3-7-J |
| 顺子 | 三张连续点数 | 8-9-10 |
| 对子 | 两张相同点数 | KK+3 |
| 散牌(高牌) | 以上都不是 | A-9-5 |

**特殊规则**:
- A23 为最小顺子
- QKA 为最大顺子
- 含大小王的牌为散牌（王不参与组合）
- 同牌型先比最大牌点数，再比花色

#### 2.3.4 花色大小

黑桃 > 红心 > 梅花 > 方块

#### 2.3.5 比牌结算

- 逐墩比较：头墩对头墩、中墩对中墩、尾墩对尾墩
- 每墩赢得 = 底分 x 1分
- 三墩全赢（通杀）额外奖励 = 底分 x 3分
- 多人对战时每两人之间互相结算

#### 2.3.6 特殊效果

- 通杀时显示"通杀"彩虹边框动画
- 被通杀时显示"被通杀"红色闪烁
- 比牌结果逐墩动画展示

### 2.4 实时通信

| 功能 | 描述 | 优先级 |
|------|------|--------|
| WebSocket连接 | 客户端与服务端实时双向通信 | P0 |
| 连接状态显示 | 右上角显示连接/断开状态 | P0 |
| 断线重连 | 断开后每2秒自动重试 | P0 |
| 房间状态同步 | 玩家列表、准备状态实时同步 | P0 |
| 摆牌进度同步 | 提示"某玩家已摆牌" | P1 |

### 2.5 数据持久化

- 用户数据持久化（用户名、密码、头像、金币、战绩）
- 房间数据持久化（服务重启后恢复）
- JSON文件存储

## 3. 非功能需求

### 3.1 性能需求

- 支持同时在线 2-6 人对战
- WebSocket 消息延迟 < 100ms（局域网环境）
- 页面首次加载 < 2s

### 3.2 兼容性

- 支持主流浏览器：Chrome、Firefox、Edge、Safari
- 支持移动端浏览器（响应式布局）
- 触屏与鼠标操作兼容

### 3.3 安全性

- 服务端发牌，防止客户端作弊
- 服务端超时强制结束摆牌
- 密码保护房间

### 3.4 可部署性

- 支持本地运行（localhost）
- 支持局域网访问（绑定0.0.0.0）
- 支持云平台部署（Render.com等，通过 `process.env.PORT` 适配）

## 4. 通信协议

### 4.1 协议格式

所有消息使用 JSON 格式，通过 WebSocket 传输。基本结构：

```json
{ "type": "消息类型", ...其他字段 }
```

### 4.2 客户端 -> 服务端 消息

| type | 描述 | 关键字段 |
|------|------|----------|
| `register` | 注册 | username, password |
| `login` | 登录 | username, password |
| `create_room` | 创建房间 | maxPlayers, password, baseScore, arrangeTime |
| `join_room` | 加入房间 | roomId, password |
| `leave_room` | 离开房间 | - |
| `toggle_ready` | 切换准备状态 | - |
| `submit_arrange` | 提交摆牌 | head[], mid[], tail[] |
| `start_next_round` | 返回房间/下一局 | goldUpdate |

### 4.3 服务端 -> 客户端 消息

| type | 描述 | 关键字段 |
|------|------|----------|
| `login_ok` | 登录成功 | username, avatarIdx, gold, isNew |
| `error` | 错误提示 | msg |
| `room_created` | 房间已创建 | roomId |
| `room_joined` | 已加入房间 | roomId |
| `left_room` | 已离开房间 | - |
| `room_state` | 房间状态同步 | room{} |
| `game_start` | 游戏开始/发牌 | hand[], players[], roundNum, baseScore, arrangeTime |
| `arrange_ok` | 摆牌确认 | - |
| `player_arranged` | 玩家已摆牌 | username |
| `all_arranged` | 所有人摆牌完成 | arranged{}, hands{}, players[] |

## 6. 技术约束

- 前端：纯 HTML + CSS + JavaScript（单文件，无框架依赖）

- 存储：JSON 文件持久化

- 无数据库依赖，无构建工具依赖
